01 — Structure des fichiers

Ce que contient le dossier

L'intégralité de la plateforme tient en 4 fichiers. Aucune dépendance à installer, aucun serveur requis pour l'ouvrir.

interface/
  index.html  ← Page d'accueil (ouvrir en premier)
  pilot-bu-dashboard.html  ← Dashboard Finance & Admin
  guide.html  ← Ce document
  data/
    kpi-output.json  ← Données calculées (générées auto)
    generate-kpi.py  ← Script de recalcul des données

Les données KPI sont intégrées directement dans les fichiers HTML via une balise <script type="application/json">. Le dashboard fonctionne donc sans serveur web, directement en double-cliquant sur index.html.

02 — Mise en place

Démarrer en 3 étapes

1

Ouvrir index.html

Double-cliquez sur interface/index.html dans votre explorateur de fichiers. Le site s'ouvre dans votre navigateur (Chrome, Edge ou Arc recommandé). Aucune installation requise.

2

Accéder au dashboard pilote

Cliquez sur « Open Pilot Dashboard » depuis la page d'accueil, ou ouvrez directement pilot-bu-dashboard.html. Le dashboard se charge avec les données Finance & Admin pré-calculées.

3

Sélectionner votre niveau d'accès

Utilisez les boutons T1 / T2 / T3 en haut à droite pour basculer entre les vues Chief Officer, BU Leadership et Manager Opérationnel. Un tour guidé se lance automatiquement à la première visite.

03 — Tiers d'accès

Trois niveaux, trois audiences

Chaque tier filtre les sections visibles du dashboard selon le profil de l'utilisateur. Le basculement est instantané via les boutons T1/T2/T3.

Tier 1
Chief Officer
COMEX · DG · DRH Groupe
KPI Scorecard BU Comparison Action Levers (détail complet) Trend & Forecast Absenteeism Analysis V_HC Composite

Présentation COMEX mensuelle. Identifier les BUs en difficulté sur l'engagement et valider les 3 leviers prioritaires. Comparer Finance & Admin avec Coverage France (benchmark naturel).

Ouvrir en Tier 1
Tier 2
BU Leadership
Directeur de BU · HR Business Partner
KPI Scorecard BU Comparison Action Levers (résumé) Trend & Forecast Absenteeism deep-dive V_HC Composite

Revue trimestrielle avec le HR BP. Comprendre le classement de la BU vs les entités comparables. Prioriser les leviers à initier et identifier les ressources RH disponibles (We Care, FAB'Life).

Ouvrir en Tier 2
Tier 3
Manager Opérationnel
Chef d'équipe · Manager de proximité
KPI Scorecard BU Comparison Action Levers Trend & Forecast Absenteeism deep-dive V_HC Composite

Point mensuel individuel avec le manager. Prendre connaissance des 5 indicateurs de l'équipe. Orientation vers les programmes RH disponibles (FAB'Life, We Care, formations certifiées) sans exposition aux données sensibles.

Ouvrir en Tier 3

05 — Mise à jour des données

Recalculer les KPIs depuis les fichiers source

Lorsque de nouveaux fichiers XLSX sont disponibles (nouvelle campagne EAE, nouvelle vague IMR, nouvelles données formation), relancez le script Python pour régénérer tous les KPIs et mettre à jour les dashboards automatiquement.

1

Prérequis : installer openpyxl

pip install openpyxl
2

Mettre les nouveaux fichiers XLSX en place

Remplacez ou complétez les fichiers source dans les 5 dossiers :

Sujet Alberthon/
  HR Data/  EAE + Data.xlsx (absentéisme)
  Training/  Training_Records, Quick_Review, Cold_Review
  Finance/  PL-FTE_22-25 (ou nouvelle version)
  Employee satisfaction & engagement/  IMR / IER PDFs
  Governance/  Bilan Social PDF
3

Lancer le script de génération

# Depuis le dossier interface/data/
python3 generate-kpi.py

Le script va :
• Lire tous les fichiers XLSX source
• Calculer les 5 KPIs pour chacune des 21 BUs
• Générer kpi-output.json
• Mettre à jour automatiquement les deux fichiers HTML (embed JSON inline)

4

Rouvrir le dashboard

Fermez et rouvrez index.html ou pilot-bu-dashboard.html dans votre navigateur. Les nouvelles valeurs s'affichent automatiquement.

Les fichiers IMR 2023/2024/2025 sont des PDFs en image (non OCRisés). Les valeurs Y de ces années utilisent les chiffres déclarés dans les rapports (3.38, 3.39). Pour extraire les données futures, un outil OCR sera nécessaire (Tesseract ou Adobe Acrobat).

06 — Onboarding par profil utilisateur

Comment former les futurs utilisateurs

Chaque profil dispose d'un tour guidé intégré (bouton « Guide » en haut à droite du dashboard). Voici le plan de montée en compétences recommandé par tier.

Tier 1 — Chief Officer

Tour guidé automatique à la première visite (6 étapes, ~90s). L'utilisateur explore les 6 sections en autonomie. Aucune formation préalable requise.

Tier 2 — BU Leadership

Tour guidé 4 étapes. Recommandé : 15 min de session avec le HR BP pour contextualiser les leviers d'action identifiés pour la BU.

Tier 3 — Ops Manager

Tour guidé 3 étapes. Présentation en point d'équipe (5 min). L'accent est mis sur la lecture des couleurs et l'orientation vers les programmes RH.

Accès direct au bon tier : Envoyez le lien incluant le paramètre tier à chaque utilisateur :

# Chief Officer (accès complet)
pilot-bu-dashboard.html?tier=1

# BU Leadership (résumé BU)
pilot-bu-dashboard.html?tier=2

# Manager Opérationnel (scorecard uniquement)
pilot-bu-dashboard.html?tier=3

Le dashboard s'ouvre directement dans la vue appropriée, et le tour guidé se lance automatiquement à la première visite.
